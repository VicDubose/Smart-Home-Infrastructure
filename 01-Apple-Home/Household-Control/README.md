# Household Control

## Overview

Apple Home is the **exclusive user-facing control layer** for the household.

All smart home interaction happens here — not in Home Assistant.

## Who Uses This

Apple Home is used in practice by:

- me
- my mom
- my dad
- other family members as needed

This makes Apple Home the shared household control layer, not just a personal dashboard.

---

## What People Control

The most important things people control from Apple Home are:

- front door lock
- AC / climate zones
- lights
- plugs and switches
- home / away state
- Eufy security visibility
- Siri voice controls
- spoken and audio-based notifications

This makes Apple Home the practical interface for everyday living rather than just a technical dashboard.

---

## Voice + Audio Experience

Apple Home enables:

- Siri voice control
- spoken alerts through HomePods
- audio notifications for household events

Examples include:

- “Laundry is complete.”
- “Dishwasher is now running during off-peak hours.”
- “Home set to Away.”

This makes the home feel responsive without requiring users to check apps manually.

## Home / Away Simplicity

<img src="../Screenshots/IMG_1323.png" width="800"/>

Apple Home makes occupancy and access easy to understand.

- shared home access
- home / away state

This same front-end state also helps coordinate other systems behind the scenes, including Eufy mode behavior and user-facing automation flow.

---

## Administrative Boundary

Home Assistant is reserved for administrators only.

Normal users should never need to interact with Home Assistant because Apple Home already provides:

- the visible controls
- the household interface
- the alerts and audio cues
- the secure remote access layer

This creates a strong separation between:

- **admin / automation backend**
- **family / household front end**

---

## Kiosk Note

A shared iPad kiosk also exists as part of the household control system.

It is part of the same front-end user experience, but its layout and behavior are documented separately in the UI/UX section.

---

## Summary

Household Control is built around a simple idea:

> family members should interact with the home, not with the automation engine

Apple Home makes that possible through:

- simple room controls
- shared access
- voice interaction
- alerts and auditory feedback
- a consistent interface across household devices
